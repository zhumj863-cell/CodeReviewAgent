# 编码规范

## 命名规范
- 变量命名使用 snake_case 风格
- 类名使用 PascalCase 风格
- 常量使用全大写加下划线，例如 MAX_RETRY_COUNT
- 函数名应清晰表达其功能，避免使用缩写
- 布尔变量应以 is_、has_、can_ 开头

## 函数规范
- 函数不超过 50 行
- 每个函数只做一件事，遵循单一职责原则
- 函数参数不超过 5 个，超过时应使用对象封装
- 必须为公共函数编写 docstring，说明参数和返回值
- 避免函数副作用，尽量使用纯函数

## 异常处理
- 必须有异常处理，禁止裸 except
- 捕获具体的异常类型，不要使用 except Exception
- 异常信息必须包含上下文，便于排查问题
- 不要用异常控制正常流程
- 关键操作需要 finally 块确保资源释放

## 变量与状态
- 禁止使用全局变量
- 避免可变默认参数，如 def func(items=[])
- 变量作用域应尽可能小
- 使用类型注解标注函数签名

## 代码结构
- 每个模块文件不超过 300 行
- import 语句按标准库、第三方库、本地模块分组排列
- 避免循环导入
- 删除未使用的代码和注释掉的代码

## 安全规范
- 禁止在代码中硬编码密码、密钥等敏感信息
- 用户输入必须进行校验和清洗
- SQL 查询必须使用参数化查询，防止注入
- 日志中禁止输出敏感数据

## 测试规范
- 单元测试覆盖率不低于 80%
- 测试函数命名格式为 test_功能_场景_预期结果
- 每个测试用例只验证一个行为
- 使用 mock 隔离外部依赖

## 日志规范
- 使用 logging 模块，禁止使用 print 调试
- 日志级别正确使用：DEBUG、INFO、WARNING、ERROR、CRITICAL
- 日志必须包含时间戳和模块信息
- 错误日志需包含完整的堆栈信息
